# Frontend Staj — 25 İş Günü

Bu repo, frontend stajım boyunca yazdığım tüm kodları, günlük notlarımı ve değerlendirme
kayıtlarını içerir. Mentörümün hazırladığı 30 günlük programı temel alır.

**Hedef:** React tabanlı frontend geliştirme
**Başlangıç durumu:** HTML/CSS var, Git/GitHub aktif kullanılıyor, JavaScript temeli zayıf
**Günlük ritim:** 6-7 saat → 1,5-2 sa teori · 3-3,5 sa pratik · 30-45 dk çıktı + not · 30 dk mentor Q&A

---

## Programın en kritik kuralı

> **React'e Gün 13'ten önce geçilmez.**
> Zayıf JavaScript temeli üzerine React öğrenmek, "çalışıyor ama neden çalıştığını bilmiyorum"
> noktasına saplanmaya yol açar. İlk 12 gün bilinçli olarak JavaScript'e ayrılmıştır.

**Gün 12 bir kontrol noktasıdır.** Gün 6-11 çıktıları eksiksiz değilse React'e geçilmez, JS
içeriği tekrarlanır. Bir haftalık gecikme, temelsiz devam etmekten daha ucuzdur.

---

## Bu plan neden 25 gün? — mentöre not

Orijinal program **30 iş günü** olarak tasarlandı. Program bana stajın 1. haftasının sonunda
ulaştığı için kalan süre **25 iş günü**. Aşağıdaki **5 birleştirme** ile 30 günlük içerik 25 güne
sığdırıldı. Hiçbir konu programdan çıkarılmadı.

Birleştirmelerin tamamı, halihazırda bildiğim ya da ön çalışmasını yaptığım alanlardan seçildi.
**JavaScript, DOM ve asenkron blokları hiç kırpılmadı** — programın tezi orada.

| Birleştirme | Orijinal | Gerekçe |
|---|:--:|---|
| Git doğrulama + Flexbox | 1 + 2 | Git ve GitHub'ı zaten aktif kullanıyorum (notlarımı commit'liyorum). Git'e tam gün gerekmedi, doğrulamaya indirildi. |
| JS temelleri I + II | 4 + 5 | Program elime ulaşana kadar geçen 4 günde JS temel tekrarına başlamıştım. Gün bir öz-testle açılıyor; zayıf çıkan konularda kalınıyor. |
| DOM I + II | 8 + 9 | İkisi de "DOM'a dokunma" pratiği ve birbirini doğrudan besliyor. Kapsam kırpılmadı, aynı güne alındı (sabah seçme/manipülasyon, öğleden sonra event'ler). |
| React Router + Styling | 23 + 24 | Styling günü zaten "önceki günün uygulamasını stillendirme" görevi. CSS tarafım güçlü; programdaki en az yeni-öğrenme içeren gün. |
| Code review + Deployment | 28 + 29 | Review bulgularını kapatma ve yayına alma art arda akan işler. |

> Bu birleştirmeler bir öneridir, mentor onayına açıktır. Onaylanmazsa alternatif: staj 26 güne
> uzar ya da farklı bir gün birleştirilir.

aynı klasörde. Böylece bir klasöre bakan kişi, dosyaları açmadan ne olduğunu anlıyor.
